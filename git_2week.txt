[git_2주차 배운 개념]

-협업하기 위한 단계는 무엇이 있을까?
1단계. 누가 이 작업 할 것인지 정한다. - Issue
2단계. 각자 맡은 것을 작업한다. - Branch
3단계. 각자 작업을 프로젝트에 합친다. - merge
(경우에 따라). 작업한 내용을 리뷰하고 최종적으로 프로젝트에 반영한다. - PR 후 merge

*작업할 브랜치로 바꾸는 것을 체크아웃(check-out) 이라고 하고, 체크아웃된 브랜치에만 commit이 반영 된다.

*브랜치의 작업 내역 commit 들을 다른 브랜치로 반영 하는 것을 merge(머지) 라고 부름. 
main 브랜치에 만들어둔 다른 브랜치들을 머지 함

* 작업이 완료되면 작업한 브랜치는 보통 삭제함. 

-merge conflict (병합 충돌) 이 발생하면 Git이 충돌을 파악하여 파일 내용을 고쳐서 충돌 내역을 보여줌
*conflict를 수정하려면 최종적으로 반영할 내역으로 고친 후 merge commit 하면 된다. 

*tracking 한다는 것은 로컬repo와 원격repo의 특정 브랜치를 연결해주는 것. 

*push와 pull은 기본적으로 tracking(추적)되고 있는 브랜치를 기준으로 commit 내역을 반영한다.
